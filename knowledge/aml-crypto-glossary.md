# AML & Crypto Glossary

> Quick reference for interviews. Expand as you study.

| Term | Definition |
|------|------------|
| **AML** | Anti-Money Laundering — laws/processes to detect illicit funds |
| **KYC** | Know Your Customer — verify customer identity |
| **CDD** | Customer Due Diligence — standard KYC for normal risk |
| **EDD** | Enhanced Due Diligence — deeper review for high-risk customers |
| **TM** | Transaction Monitoring — rules/models on transaction flows |
| **STR / SAR** | Suspicious Transaction/Activity Report filed to authorities |
| **VASP** | Virtual Asset Service Provider (FATF term for exchanges etc.) |
| **FATF** | Financial Action Task Force — global AML standard setter |
| **Travel Rule** | VASP must share originator/beneficiary info on transfers |
| **PEP** | Politically Exposed Person — higher risk, EDD often required |
| **Sanctions screening** | Match customers/tx against OFAC/UN/EU lists |
| **Typology** | Pattern of illicit behavior (ATO, layering, mule, etc.) |
| **False positive (FP)** | Alert that is not actual fraud/AML after review |
| **True positive (TP)** | Alert confirmed as fraud/AML |
| **Hit rate** | Alerts triggered ÷ eligible events |
| **Layering** | Moving funds through multiple hops to obscure origin |
| **Structuring** | Breaking amounts below reporting thresholds |
| **Money mule** | Account used to pass through illicit funds |
| **ATO** | Account Takeover — unauthorized access |
| **Onboarding fraud** | Fake/synthetic identity at registration |
| **Bonus abuse** | Exploiting promotions/referrals |
| **P2P scam** | Fraud via peer-to-peer fiat/crypto trades |
| **KYT** | Know Your Transaction — blockchain tx screening (Chainalysis etc.) |
| **Blockchain analytics** | Tracing wallets/entities on-chain |
| **Wallet clustering** | Grouping addresses to same entity |
| **Mixer / tumbler** | Service obscuring transaction trail |
| **Cold / hot wallet** | Offline vs online exchange storage |
| **Fiat on-ramp** | Converting fiat → crypto (card, bank) |
| **Off-ramp** | Crypto → fiat withdrawal |
| **STR filing deadline** | Jurisdiction-specific (know conceptually) |
| **RBA** | Risk-Based Approach — controls scaled to risk |
| **MLRO** | Money Laundering Reporting Officer |
| **2LOD / 3LOD** | Second/third line of defense (audit/compliance) |
| **Case management** | Workflow from alert → investigation → disposition |
| **Disposition** | Outcome: TP, FP, escalate, SAR |
| **Rule tuning** | Adjust thresholds/logic to balance FP vs TP |
| **Model score** | ML probability added to rules |
| **Whitelist** | Pre-approved low-risk entities |
| **Grey list** | FATF jurisdictions under increased monitoring |
| **OSINT** | Open-source intelligence in investigations |
| **Chain hopping** | Moving assets across chains to evade detection |
| **DeFi** | Decentralized finance — new AML exposure |
| **Smart contract** | On-chain programmable logic |
| **Stablecoin** | Token pegged to fiat (USDT, USDC) |
| **Proof of reserves** | Exchange attestation of asset backing |
| **CCAS** | ACAMS Certified Cryptoasset AFC Specialist |
| **CAMS** | Certified Anti-Money Laundering Specialist |
| **GoAML** | UNODC reporting system used in many jurisdictions |
| **MAS** | Monetary Authority of Singapore |
| **SFC** | Securities and Futures Commission (Hong Kong) |
| **FIU** | Financial Intelligence Unit — receives STRs |
